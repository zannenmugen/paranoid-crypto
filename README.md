# Project Paranoid

## Overview

**Paranoid** is a project that checks for well-known weaknesses in cryptographic artifacts such as public keys, digital signatures, and general pseudorandom numbers. This library contains implementations and optimizations of existing techniques from the literature. The motivation stems from cases where the generation of these artifacts has been flawed.

## Goal

The goal is to increase confidence in cryptography use cases both inside and outside organizations.

When dealing with asymmetric encryption, cryptographic artifacts usually are:

1. Generated by internal tools, or  
2. Generated by third-party tools that can be checked for vulnerabilities, or  
3. **Generated by third-party tools and/or hardware or software black boxes that cannot be inspected.**

Paranoid can test any cryptographic artifact but is primarily motivated to detect weaknesses in third-party black boxes. It can be used even when the source code is not accessible.  

The project aims to detect both known and unknown vulnerabilities. Detecting known vulnerabilities is straightforward, while identifying new vulnerabilities may require large sets of artifacts or may detect weak artifacts only with low probability. Feedback and contributions regarding weak cryptographic implementations are encouraged. The project is continuously evolving, and new detection methods are added as weaknesses are discovered.

## Getting Started
